# neovim キーバインド チートシート

> VSCodeから移行した人向けに、「何をしたいか」ベースで整理しています。

## はじめに：モードの概念（VSCodeにない概念）

nvimにはモードがあります。起動直後は **ノーマルモード** です。

| モード | 入り方 | 何ができる |
|---|---|---|
| ノーマルモード | `Esc` または `jk` | ナビゲーション・コマンド実行（ここが基本） |
| インサートモード | `i` | テキスト入力（VSCodeの通常状態に近い） |
| ビジュアルモード | `v` | テキスト選択 |

**Leaderキー = `Space`**（以降 `<leader>` と表記）

## ファイルを開く・探す

| やりたいこと | キー | VSCodeの対応 |
|---|---|---|
| ファイル名で検索して開く | `<leader>sf` | `Ctrl+P` |
| ファイル名で検索（Telescope） | `<leader>ff` | `Ctrl+P` |
| Gitで管理されているファイルを検索 | `<leader>gs` | `Ctrl+P`（.gitignore対応版） |
| ファイルエクスプローラーを開く（親ディレクトリ） | `-` | サイドバーのエクスプローラー |
| 開いているファイル（バッファ）一覧 | `<leader>fb` | `Ctrl+Tab` |
| 関連ファイルに切り替え（実装 ↔ テストなど） | `<leader>oo` | なし |
| 関連ファイルを新しいタブで開く | `<leader>ot` | なし |

## コードを読む・定義へジャンプ

| やりたいこと | キー | VSCodeの対応 |
|---|---|---|
| 定義へジャンプ | `gd` | `F12` |
| 宣言へジャンプ | `gD` | `F12`（宣言） |
| 実装へジャンプ | `gi` | `Ctrl+F12` |
| 参照一覧を表示 | `gr` | `Shift+F12` |
| ホバーでドキュメント表示 | `K` | カーソルを乗せる |
| カーソル下と同じ変数・単語の次の参照へ | `]]` | なし |
| カーソル下と同じ変数・単語の前の参照へ | `[[` | なし |

| やりたいこと | キー | VSCodeの対応 |

## ターミナル・Git

| やりたいこと | キー | VSCodeの対応 |
|---|---|---|
| ターミナルを開閉する | `Ctrl+\` | `` Ctrl+` `` |
| Lazygitを開く（フロート） | `Ctrl+G` | Source Controlパネル |
| Lazygitを開く（別方法） | `<leader>gg` | Source Controlパネル |
| 現在のファイルをGitHubで開く | `<leader>gf` | なし |

| GitHubリポジトリのトップを開く | `<leader>gr` | なし |
| GitHubでブラウズ（ブランチ選択あり） | `<leader>gB` | なし |
